University-related projects

Please note that the data files are not available and the links only provide codes and/or reports

  • Analysis of GS1: 1-Year Treasury Constant Maturity Rate, link to paper

    Individual project aimed at performing an analysis of the given time series and producing one-step-ahead forecasts. The analysis includes a decomposition of the time series, a study of the ACF and PACF, and the selection & validation of ARIMA models with GARCH effects.

  • Correlation Between SHEL price & Other Securities, link to paper (R code upon request)

    Application of statistical models seen in class to explore correlations between the price of Shell's stock and other financial securities.

  • Pilot Machine Learning project, link to code

    Individual project aimed at building an end-to-end pipeline for sample data and apply ML techniques to perform a multiclass classification task.

  • Gene-based classification of Hypoxia in cancer cells, link to code

    Group project in which various ML techniques are explored to classify the state of hypoxia in cells. The study analyzes single-cell sequencing data from two cancer cell lines (HCC1806 and MCF7) using Smartseq and Dropseq techniques. In the last part, an additional model aimed at producing a more general classifier is explored. In fact, building a satisfactory multi-cell lines &/or multi-sequencing techniques classifier is under current research.

  • A convex optimization problem with EV cars, link to code

    Used cvxpy to solve a model problem on the efficiency of Electric cars, given parameters on battery, fuel consumption and so on.

  • Big-M Method to solve mixed-ILP, link to code

    It is a method that allows to solve ill-posed LP problems by introducing additional pairwise mutually exclusive constraints that are "turned on/off" through binary variables.

  • Implementation of Earth Mover's Distance in basic object recognition, link to code

    Find the pattern of motion of an object given some frames. The Earth Mover's distance (a.k.a. Wasserstein Metric) is implemented by running a min-cost flow problem on bipartite graphs built pairwise between the frames.
